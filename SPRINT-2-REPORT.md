## What's New (User Facing)

- Gemini API LLM has been swapped out with LLaMA 3.2 1B Instruct.
- The large language model can be easily be swapped out for other models on Huggingface by setting the LANGUAGE_MODEL_ID in a .env file. The default is LLaMA 3.2 1B Instruct. This should work best with swapping to newer or more powerful LLaMA models.
- Uploading PDFs for the AI to use in learning is now supported.
- You can now start new sessions with the AI tutor or retrieve old sessions.
- The system can run entirely locally after the first run. It needs to connect to Huggingface once to obtain the language model parameters and tensors, but can be ran locally after.
- The UI now fully supports quizzes generated by the AI. It will give a list of multiple-choice questions and you will be given a score and a list of correct answers after.

## Work Summary (Developer Facing)

The entire backend has been reworked to accommodate local LLMs. In the sprint 1 prototype, we were using the online Gemini API, which retrieved answers from Google's servers rather than running locally. This has been changed. To help make this process easier, we have heavily reworked the system prompt to be more friendly to local LLMs, specifically, LLaMA. However, we plan to allow this to be modified in the future by a school admin if necessary for a different model. The model should run on most high-end systems and is ideally ran on a system with 13+ GB of VRAM. RAG (Retrieval-Augmented Generation) has been implemented, which is what powers the ability for the AI to learn from uploaded files. On the front-end, we made some changes to the system so that it is no longer just an AI chatbot and instead can actually accept the PDFs for RAG and will save prior chats for the user to go back to in the future.

## Unfinished Work

US-03 and US-05 are still being worked on. US-03 is viewing AI tutor usage reports to evaluation adoption and effectiveness across classrooms. This will be done in the next sprint when the full login and role-based access system is implemented, as well as the classroom monitoring systems. We currently are storing the user sessions, but they are not being shared with school admins or teachers yet.

US-05 is about adapting question difficulty to receive personalized guidance for students. This is quite difficult and will be tied heavily to our mastery tracking system that is planned to be the primary focus of Sprint 3.

Functional requirements 1, 2, 6, and 8 have not been started on. Functional requirements 4, 5, 7, and 9 have already had meaningful progress but will not be ready for Sprint 2. Most of them are related to the mastery tracking system and the monitoring system, both of which will be fully implemented in Sprint 3.

## Retrospective Summary

Here's what went well:

- Swift implementation of features after schedule pivot in last meeting
- More evenly divided work during this sprint than last time
- More experience with language models allowed us to work with the local LLM despite not being as advanced as Gemini was
- We began meeting with the client every 2 weeks rather than once a week due to the weekly meeting resulting in not having enough to talk about to even write down a meeting minutes

Here's what we'd like to improve:

- We need to complete more of the functional requirements.
- We had to pivot from what our original plan for this sprint was (Focus on mastery tracking) to complete other parts of the program. This resulted in less time put into things than we planned for.
- Due to hardware limitations, we have been unable to test a more powerful language model. The client has offered a school district server for us to test with that has more VRAM and RAM, so we plan to use it to test the tutoring application with a 7B or larger parameter LLM using the same 128k context window or bigger.

Here are changes we plan to implement in the next sprint:

- We will use the school district server with a more advanced LLM for testing the mastery tracking system.
- We will fully plan out the weeks of this sprint so that each feature is given the time they require.
- The functional requirements will all be completed by next sprint.
